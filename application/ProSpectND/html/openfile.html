Open and Read Files

The program can only read data that are stored in its own internal format. Therefore, NMR data first has to be converted before it can be processed. A couple of scripts do exist, to convert Bruker and Varian files to this internal format. The internal format stores the data in the form of sub cubes what makes the processing of multi dimensional data more efficient. Little- or bigendianness of files is detected automatically and taken care of.

Open File

    Command line: GE <filename>
    Script: ropen <filename>

    Use File->Open File to select a NMR file for input. A 1D file will be loaded immediately. For a multi dimensional NMR file, the user is prompted for a direction to read and a record number.

Close File

Script: rclose

Input files are closed automatically when a new file is opend, or when the program is terminated. Use only in special case.

Set Direction

    Command line: DR <direction x> { <direction y> { <direction ...> } }
    Script: DR <direction x> { , <direction y> { , <direction ...> } }

    Set the direction(s) (or file access) for a multi dimensional NMR experiment. Use File->Change Direction or 

    Set file access:

    Set the direction(s and plane offset) for a multi dimensional NMR experiment. For 2D files: DR <direction> Where <direction> can be 1 or 2. In a 2D plot, this direction is used for the x axis and the other one for the y axis. For 3D files: DR <direction 1> <direction 2>  Where <direction 1> is the processing direction, and <direction 2> determines together with <direction 1> what planes are used.
     
     

Read Record

    Command line: RN <record number> { <record number> }
    Script: RN { <record number> } {, <record number> } {, <range> }

    Read new record from file. Use File->Read Record or 

    1D file: record is loaded inmediately.
    2D file: asks for record number

    With the  buttons, one can select the next or the previous record in a plane. 3D file: asks for 2 record numbers. The first is the 2D record number and the second one is the plane offset.

Return Record

Command line: RTR
Script: RTR

Return a (modified) record to the same file it was read from.

Write Record

Command line: OU <filename>
Script: WR @C, "filename"

Write the current record to a new 1D file.

Change Directory

Command line: CHDIR <filename>
Script: CHDIR "filename"

Change the current working directory.
