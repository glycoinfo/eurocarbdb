The command line

Most commands are available by menus and/or buttons as well as via the command line.  In the first case, the user is prompted for input via popup menus, while in the second case parameters can be given with the command or current parameters are used silently. Also, some parameters can be set via the command line, mostly for historic reasons.

File

GE <filename> { <direction> { <direction> ... } }

Open the file <filename> for input.

CL

Close the current input file.

DR <direction x> { <direction y> { <direction ..> } }

Set the direction(s and plane offset) for a multi dimensional NMR experiment. For 2D files: DR <direction> Where <direction> can be 1 or 2. In a 2D plot, this direction is used for the x axis and the other one for the y axis. For 3D files: DR <direction 1> <direction 2>  Where <direction 1> is the processing direction, and <direction 2> determines together with <direction 1> what planes are used.

RN <number> { <number> }

Read new record from file.

RTR

Return current (modified) record to file.

OU <filename>

Save current record to the file <filename>

CHDIR <dirname>

Change current working directory to <dirname>

Parameters

SP

Spectral Parameters. Set the parameters SF, SW, XR and File Record type.

SF <frequency>

Set spectrometer frequency in MHz.

SW <frequency>

Set spectral width in Hz.

XR <channel>

Set reference channel for ppm calibration. PPM is set to 0 at this channel.

RM

RM stands for Read Mode and is used to set the pre-processing parameters Read Mode, Read Start and Read Stop. For a file with complex data, Read Mode determines how this data is loaded. The options are real (only real data is loaded), i/r swapped (real data is stored as imaginary end vice versa) and complex. In most cases Read Start = 1 and Read Stop is equal to the record size. These parameters can be used to read only a selective part of records from file.

DSP

DSP Mode.

LM

Linear prediction mode.

WM

Window Mode.

SH <shift>

Set shift (theta) for KS, SN and SQ.
 

LB <line broadening>

Set line broadening for EM, CD and GM.
 

GB <fraction>

Set Gaussian broadening (fraction) for GM
 

IT <number of points>

Set number of points for Trapezian multiplication.

WW

Watwa mode.

FM

Fourier Mode

ZF

Set the number of zero fill.

FTSCALE

Set scaling of first data point before Fourier transformation.

HB

Hilbert Transform mode.

PM

Phase Mode.

PA <phase0>

Set the frequency independent phase. <phase0> is the value of the frequency independent phase.

PB <phase1>

Set the frequency dependent phase. <phase1> is the value of the frequency dependent phase.

I0 <i0>

Set the frequency independent position. <i0> is the frequency independent position in channel numbers.

XM

Exchange Mode.

BM

Baseline Mode.

SM

Store Mode.

ND

Next processing direction
 

EDP

Edit all Parameters.

ST

Store Parameters.

GP

Get parameters.

CH

Change Parameters.

CM

Contour Mode.

Show Parameters and data

    LP

    List current parameters in the current processing direction

    LA

    List all parameters form input file

    LAC

    List all current parameters

    HI

    List history files (scipts)

    AQ

    List original acquisition parameters


Processing

    SI <size>

    Set data size.

    FZ <times zero fill>

    Fill Zero. Enlarge the spectrum <times zero fill> * <data size> and set the new part to zero. This is done automatically by the FT routine. However, DFT does not. So is usefull in combination with DFT.

    AV

    Absolute Value spectrum.

    PS

    Power Spectrum

    AU

    Automatic processing for all options that are set. Options are: Linear prediction, Window, Watwa, Fourier transform, Hilbert transform, Phase correction, Reverse, Baseline correction.

Water Wash

    WATWA <power> <convolution width> { <shift> }

    Water Wash. Filter large peaks (waterline) from spectrum <power> Filter is of the form cos^<power> <convolution width> 64 seems to be OK. <shift> Number of channels to shift peak to the middle of the spectrum.

Windows (Filter Functions)

    HM

    Hamming window

    HN

    Hanning window

    SN { <shift> }

    Sine bell<shift> Typical values: [3..10] for 2D NOE, 2D HOHAHA. [1000] pure sine, for absolute value COSY

    SQ { <shift> }

    Squared sine bell <shift> typical values: [3..10] for 2D NOE, 2D HOHAHA. [1000] pure sine, for absolute value COSY

    EM { <line broadening> }

    Exponential multiplication <line broadening> is the line broadening in Hz.

    GM { <linewidth> { <fraction> } }

    Lorentz-gauss transformation <linewidth> the linewidth (in Hz). <fraction> fraction of the data size.

    CD { <line broadening> }

    Convolution difference <line broadening> is the line broadening in Hz.

    TM { <number of points> }

    Trapezian multiplication

    KS { <theta> }

    Kaiser window<theta> typical values: [1.5..3] If <theta> is not supplied, a popup menu is activated that asks for <theta>.

    WB

    Window from buffer. Use the window function that has been prepared in Buffer A.

Fourier Transformation

    FT { <zero fill> }

    Fourier transformation. <zero fill> gives the number of zero fills before Fourier transformation. If <zero fill> is not specified, ZF is assumed.
     

    DFT

    Discrete Fourier transform. Data size does not have to be a power of two. See also FZ.

    IFT

    Inverse Fourier transformation.

Phase

    PK <phase0> <phase1> <i0>

    Perform phase correction with <phase0>, <phase1> and <i0>. Previously defined parameters are used when not given with this command. Parameters can be determined interactively. <phase0> Frequency independent phase <phase1> Frequency dependent phase <i0> Frequency independent position

    PZ

    Reset phase parameters. Phase0 = 0, phase1 = 0, and i0 = 1.
     

    PN

    Normalize phase parameters. Recalculate the parameters with I0 set to 1. This results in parameters that are not dependent on the size of the spectrum.

Hilbert Transformation

    HT

    Restore imaginary data before doing a phase correction.

Baseline correction

    BC

    Do baseline correction with current parameters.

Water fit

WATERFIT { <position> { , <width> } }

Waterline fit.


Set Axis Units

    Set axis units to Channels, PPM, Herz or Seconds. Set x-axis units for 1D files or x- and y-axis units for 2D files.

    CHAN

    Set axis units to channels.

    PPM

    Set axis unist to ppm.

    HZ

    Set axis units to Herz.

    SEC

    Set axis units to seconds.

    Toggle the display of data arrays on or off.

    RE

    Toggle real data on/off.

    IM

    Toggle imaginary data on/off.

    BA

    Toggle buffer A on/off.

    BB

    Toggle buffer B on/off.

    WD

    Toggle window data on/off.

    BL

    Toggle baseline on/off.

    Switch to block by typing its block number 1, 2, 3 or 4.

Move data arrays

MV

Move (=Copy) array to another array in the same block.

WR

Write array to disk.

RD

Read array from disk (but not the parameters).

Fill data arrays

    ZE

    Set real + imaginary data arrays to zero.

    ZR <value>

    Set real array to value

    ZI <value>

    Set imaginar array to value

    ZA <value>

    Set buffer A to value

    ZB <value>

    Set buffer B to value

    ZW <value>

    Set window buffer to value

    ZL <value>

    Set baseline buffer to value

Shift data in arrays

IC <clip start> <clip stop>

Clip real and imaginary data arrays from <clip start> to (and including) <clip stop>. Unit is channels.

LS  <number of points>

Left shift. All elements in the real and imaginary data arrays are shifted <number of points> positions to the left. Array elements shifted outside array boundaries are lost. Array elements shifted from outside array boundaries are substituted by 0.0. If  <number of points> is not supplied, it is set to 1. See also, in scripts: Left shift operator "<<"

RS <number of points>

Right shift. All elements in the real and imaginary data arrays are shifted <number of points> positions to the right. Array elements shifted outside array boundaries are lost. Array elements shifted from outside array boundaries are substituted by 0.0. If  <number of points> is not supplied, it is set to 1. See also, in scripts: Right shift operator ">>"

ROL <number of points>

Rotate left. All elements in the real and imaginary data arrays are circular shifted <number of points> positions to the left. Elements that are shifted below the first position are added from the right. If  <number of points> is not supplied, it is set to 1. See also, in scripts: Rotate Left operator "|<"

ROR <number of points>

Rotate right. All elements in the real and imaginary data arrays are circular shifted <number of points> positions to the right. Elements that are shifted past the last position are added from the left. If  <number of points> is not supplied, it is set to 1. See also, in scripts: Rotate Right operator ">|"
 

RV

Reverse real and imaginary data arrays. First point becomes last point, etc.

NM

Negate real and imaginary data arrays. Positive becomes negative and vice versa.

RUN { <scriptname> { <arguments> } }

    Run a script. If RUN is used without arguments, the script that is currently in the editor is used. <scriptname> The name of the script to run. <arguments> Arguments to the script to run. Example: 
            run dophase.au 123.4 44.1 102
